Commands

Filename
Syntax: Filename(name)
Argument types: name:string
Meaning: Sets file name for the generated image.
Usage example: 
Filename("output")
Import
Syntax: Import(packageName)
Argument types: packageName:string
Meaning: Imports the specified package. After a package is imported, functions and variables from it can be used without being prefixed with the package name. The prefix is still needed in case of a conflict (e.g. two functions from different packages having the same name).
Usage example: 
Import("shape")
Load
Syntax: Load(variable, filename)
Argument types: variable:variable name
filename:string
Meaning: Loads an image from the specified file and stores it as a generator in a new variable.
Note: the variable must not have any value before this command is used.
Usage example: 
Load($input, "input.png")
Randomize
Syntax: Randomize(doRandomize)
Argument types: doRandomize:boolean (0 or 1)
Meaning: Enables or disables randomization. By default, filters with the same seed and arguments always generate the same output. Enabling randomization changes this behavior, so that the generation is fully randomized.
Usage example: 
Randomize($TRUE)
Size
Syntax: Size(size)
Argument types: size:integer number between 1 and 8192
Meaning: Sets size (in pixels) for the output image.
Usage example: 
Size(512)
Smoothing
Syntax: Smoothing(level)
Argument types: level:0, 1 or 2
Meaning: Sets texture smoothing (antialiasing) level. 0 means no smoothing, higher values will slow down image generation.
Usage example: 
Smoothing(2)
Texture
Syntax: Texture(generator)
Argument types: generator:any generator
Meaning: Generates image with the currently set size and saves it in a file with the currently set file name in PNG format.
Usage example: 
Texture(
    perlin.noise:20(
        1.5,
        color.red(),
        color.yellow()
    )
)
Video
Syntax: Video(frames, generator)
Argument types: frames:integer number between 1 and 7,500
generator:any generator
Meaning: Like Texture, but generates multiple images/frames. First argument determines how many frames should be generated, filters from the video package can be used to change the output according to the current frame/time.
Usage example: 
Video(
    100,
    transform.rotate(
        shape.circle(1, 0, 0.5, 0.2, 0.2),
        0.5, 0.5, video.time()
    )
)
Debugtex
Syntax: Debugtex(size)
Argument types: size:integer number between 1 and 8192
Meaning: Generates image for each currently defined custom variable, all images will have the specified size.
Usage example: 
Debugtex(256)
Debugtime
Syntax: Debugtime(measureTime)
Argument types: measureTime:boolean (0 or 1)
Meaning: Enables or disables time measurement. When enabled, every Texture and Video command will measure the time it takes to generate each image/frame.
Usage example: 
Debugtime($TRUE)
